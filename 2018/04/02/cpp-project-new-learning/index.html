PS: 题外话

C与C++语言中，volatile关键字意图允许内存映射的I/O操作。这要求编译器对此的数据读写按照程序中的先后顺序执行，不能对volatile 内存的读写重排序。因此关键字 volatile 并不保证是一个内存屏障。
对于Visual Studio 2003，编译器保证对 volatile 的操作是有序的，但是不能保证处理器的乱序执行
